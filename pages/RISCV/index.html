Building a Single-Cycle RISC-V Processor from Scratch in Verilog HDL

May 17, 2026

I. Introduction

Abstractions are useful, but they hide the true beauty of computer architecture. This post details the construction of a 32-bit (RV32I) single-cycle RISC-V processor entirely in Verilog HDL, peeling back the layers to understand how software instructions map to physical hardware signals.

II. Understanding the RISC-V Instruction Set Architecture (ISA)

The RISC-V base integer instruction set relies on a highly regular 32-bit instruction format. We will decode the six fundamental formats: R-type, I-type, S-type, B-type, U-type, and J-type. The machine parses the opcode, funct3, and funct7 fields to route data through the hardware.

III. Designing the Datapath (The Modular Approach)

A processor is a collection of interacting modules. Here is the breakdown of the datapath:

  • Program Counter (PC) & Instruction Memory: The instruction fetch mechanism.
  • Register File: 32x32-bit registers with dual asynchronous reads and synchronous writes.
  • The ALU: Executing ADD, SUB, AND, OR, XOR, and bitwise shifts.
  • Data Memory: Interfacing with Load (LW) and Store (SW) operations.
  • Immediate Generation: Sign-extending variable immediate fields.

IV. The Control Unit (The Brains)

While the datapath moves the bits, the Control Unit tells them where to go. By mapping the opcode to multiplexer select lines (e.g., ALUSrc, MemtoReg, RegWrite), the control unit orchestrates the entire cycle.

V. Verilog Implementation Details

Transitioning from block diagrams to RTL requires strict adherence to synchronous design principles. Here is an example of implementing the ALU using combinational logic:

module ALU (
    input  [31:0] a, b,
    input  [3:0]  alu_control,
    output reg [31:0] result,
    output zero
);
    assign zero = (result == 0);

    always @(*) begin
        case (alu_control)
            4'b0000: result = a & b; // AND
            4'b0001: result = a | b; // OR
            4'b0010: result = a + b; // ADD
            4'b0110: result = a - b; // SUB
            default: result = 32'b0;
        endcase
    end
endmodule

VI. Simulation, Verification, and Visualization

Writing the RTL is only half the battle. We verify the processor by loading machine code compiled from assembly into our instruction memory and observing the waveform outputs to track the propagation of signals through the datapath.

VII. Conclusion & Future Scope

A single-cycle processor is structurally simple but suffers from a long critical path. The next evolution in this design is pipelining, introducing pipeline registers to handle data and control hazards.
